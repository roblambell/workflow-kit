item: H-SHB-2
date: 2026-04-03T11:31:33Z
summary: Canonicalize stored repoRef values to the repo hash
context: The work item required shared handling for repoUrl, repoHash, and stored repoRef inputs, but did not specify whether stored repoRef data could only be a hash or might include legacy raw values.
decision: resolveRepoRef treats repoRef as the persisted comparison value and accepts either a normalized or URL-like raw repo reference or an existing SHA-256 hash, always returning the canonical hash.
rationale: This keeps one comparison primitive for new broker data while remaining tolerant of pre-hash or manually persisted repoRef values during rollout.
