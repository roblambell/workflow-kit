date: 2026-04-12T09:06:00Z
summary: Persist dismissed update state as skipped_update_version and expose updater commands as executable+args metadata
context: The work item required install-aware update runtime state and persistence for a dismissed release, but it did not prescribe the persisted field name or the shape of the reusable command metadata needed by later update flows.
decision: Use skipped_update_version in ~/.ninthwave/config.json and represent supported updater commands as executable, args, and display metadata attached to the passive update state via resolveCurrentInstall().
rationale: The config key is explicit and narrowly scoped to the startup prompt use case, while executable/args/display metadata can be reused by the upcoming update command and startup prompt without introducing shell-specific parsing for every caller.
