item: H-CF-3
date: 2026-04-10T20:12:06Z
severity: medium
description: After PR #630 was created, the worker did start `nw inbox --wait H-CF-3`, but it then behaved like an interactive assistant instead of staying in the non-interactive inbox loop required by the implementer contract. The orchestrator's review message was present and got surfaced only when the wait session was manually polled later, after a user prompt. After that first message, a subsequent required `nw inbox --check H-CF-3` returned a CI fix request, but that check was not treated as a mandatory drain-and-switch boundary before local testing continued. Impact: pending orchestrator work was not acted on autonomously, review/CI follow-up was delayed, and the human had to prompt the worker back into the inbox-driven loop. Likely failure mode: once the worker reached post-PR wait mode, the turn-level interaction model still allowed a user-facing final response path, so inbox waiting became observational instead of authoritative. A useful fix would make post-PR wait mode exclusive until an inbox message is drained, and make any non-empty `nw inbox --check` result immediately preempt the current subtask rather than being treated as advisory output.
