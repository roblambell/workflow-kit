item: H-UPD-1
date: 2026-04-12T10:12:00Z
severity: medium
description: Implementer workflow does not currently document a convention for how to react to PR comments that the reviewer marks with a 👀 (eyes) reaction. In this PR, the user added 👀 reactions on several PR review comments and an inline comment (including "update the friction log so it has that context, also, you should reply directly to pr commends when the 👀 emoji has been added"), signalling that those comments expected a threaded reply directly on the comment rather than a top-level PR comment. The current `agents/implementer.md` "Review Feedback" handler only says to "Post a reply on the PR" -- it does not distinguish between top-level PR comments and threaded replies on specific review comments, and it does not mention the 👀 reaction as a signal at all. Recommendation: extend the implementer post-PR contract so that any PR comment with a 👀 reaction from a trusted collaborator is treated as "reply to this comment in-thread" (using `gh api repos/{owner}/{repo}/pulls/{pr}/comments --field in_reply_to=<comment_id>` for review comments, or `gh api repos/{owner}/{repo}/issues/comments/{id}/reactions` acknowledgments for issue comments), and that the orchestrator inbox surfaces the specific eyes-marked comment IDs so the worker does not have to scrape reactions manually.
